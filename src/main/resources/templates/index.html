📊 HTTP Monitoring

All HTTP requests made by this application are monitored by the server-agent for tracking request counts, types, and response times.

🔍 No WebSocket Issues

This demo uses pure HTTP/REST instead of WebSocket, avoiding the WebSocket compatibility issues encountered with React.

⚡ Performance Tracking

Built-in endpoints provide CPU, memory, and action metrics for real-time performance monitoring.

📈 Real-time Data

Trigger test actions and view logs in the dashboard to see how the agent captures and reports request metrics.

Quick Actions (for testing server-agent monitoring)

📝 How to use with server-agent:

1. Start this application on port 5173: mvn spring-boot:run

2. Configure server-agent to monitor this host (default: ::1:5173)

3. Click buttons above to trigger HTTP requests

4. Check server-agent logs to see request interception and metrics collection
